# Universal Prompts Folder README

এই ফোল্ডারে থাকা প্রতিটি প্রম্পট ফাইলের উদ্দেশ্য, কখন ব্যবহার করবেন, এবং কী ফলাফল পাবেন তা নিচে ব্যাখ্যা করা হলো:

---

## master_orchestration_prompt.md
**উদ্দেশ্য:**
- যেকোনো প্রকল্পের জন্য সম্পূর্ণ ডকুমেন্টেশন সেট জেনারেট করার একমাত্র রিইউজেবল প্রম্পট।
- সব রোল (ফ্রন্টএন্ডসহ) ধারাবাহিকভাবে চালায়।

**কখন ব্যবহার করবেন:**
- নতুন কোনো প্রকল্পের ডকুমেন্টেশন জেনারেট করতে চাইলে, এই প্রম্পটটি ব্যবহার করুন।

**ফলাফল:**
- DOC/PROJECT PLAN/ ফোল্ডারে সম্পূর্ণ, ইমপ্লিমেন্টেশন-রেডি ডকুমেন্টেশন তৈরি হবে।

---

## Continue_Role_Orchestration_Prompt.md
**উদ্দেশ্য:**
- যখন ফ্রন্টএন্ড ডকুমেন্টেশন আগেই সম্পন্ন, তখন বাকি রোলগুলো ধারাবাহিকভাবে চালানোর জন্য।

**কখন ব্যবহার করবেন:**
- যদি ফ্রন্টএন্ড অংশ আগে থেকেই তৈরি থাকে এবং শুধু বাকি ডকুমেন্টেশন দরকার হয়, তখন এই প্রম্পটটি ব্যবহার করুন।

**ফলাফল:**
- DOC/PROJECT PLAN/ ফোল্ডারে ফ্রন্টএন্ড বাদে বাকি সব ডকুমেন্টেশন তৈরি হবে এবং ফ্রন্টএন্ড অংশ অপরিবর্তিত থাকবে।

---

## master_execution_prompt.md
**উদ্দেশ্য:**
- ডকুমেন্টেশন তৈরি শেষ হলে, ইমপ্লিমেন্টেশন (কোড লেখা) শুরু করার জন্য একমাত্র প্রম্পট।

**কখন ব্যবহার করবেন:**
- যখন ডকুমেন্টেশন সম্পূর্ণ এবং এখন কোড ইমপ্লিমেন্ট করতে হবে, তখন এই প্রম্পটটি ব্যবহার করুন।

**ফলাফল:**
- ডকুমেন্টেশন অনুযায়ী কোড ইমপ্লিমেন্টেশন শুরু হবে।

---

## Execution Constitution.md
**উদ্দেশ্য:**
- ডকুমেন্টেশন ও ইমপ্লিমেন্টেশন উভয় পর্যায়ে কঠোর নিয়ম ও সিকোয়েন্স নিশ্চিত করে।

**কখন ব্যবহার করবেন:**
- প্রতিটি ধাপে নিয়ম, অর্ডার, ও কোয়ালিটি গেট নিশ্চিত করতে এই ফাইলটি পড়া বাধ্যতামূলক।

**ফলাফল:**
- পুরো ওয়ার্কফ্লো সঠিকভাবে, নির্ভুলভাবে এবং ধারাবাহিকভাবে সম্পন্ন হয়।

---

**নোট:**
- প্রতিটি প্রম্পট নির্দিষ্ট পর্যায়ে একবারই ব্যবহার করুন।
- ভুল প্রম্পট বা ভুল পর্যায়ে ব্যবহার করলে ফলাফল সঠিক হবে না।
