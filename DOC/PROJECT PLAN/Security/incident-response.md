# Incident Response

## Severity Priorities

- P0: exposure of inquiry data or admin takeover.
- P1: loss of contact submission capability or malicious public content publication.
- P2: degraded analytics, notifications, or non-critical content workflows.

## Response Flow

1. Detect and classify the incident.
2. Contain access or system impact.
3. Preserve relevant logs and audit evidence.
4. Restore safe operation.
5. Document root cause and corrective actions.

## Immediate Actions

- Disable affected admin access.
- Rotate exposed secrets.
- Verify public pages, contact flow, and content integrity after containment.
