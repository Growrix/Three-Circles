# Component And Styling Standards

## Visual Direction

The UI should feel editorial rather than templated. It should pair clean structure with strong imagery and confident typography so the site looks closer to a design studio portfolio than a generic service brochure.

## Color Tokens

- Primary accent: `#f90000`
- Primary text / dark surface: `#231f20`
- Primary light surface: `#ffffff`
- Light support surface: `#f5f5f5`
- Dark support surface: `#18181a`
- Muted border / secondary text: `#b0b0b0`

## Theme Rules

- Light mode is the default presentation.
- Dark mode is supported because the source plan defines both theme variants.
- Accent red is reserved for actions, highlighted headings, and focused interactive states.

## Typography Direction

- Use a high-contrast editorial heading face paired with a clean sans-serif body face.
- Headlines should be concise and spacious.
- Body copy should remain readable and professional rather than decorative.

## Component Inventory

- Global header and footer.
- Hero block.
- Service teaser cards.
- Portfolio grid cards.
- Portfolio detail gallery module.
- Testimonial or trust-signal band.
- CTA banner.
- Structured contact form.

## Interaction Rules

- Motion should be restrained and purposeful: image reveals, staggered section entry, and hover emphasis on portfolio items.
- Buttons require strong contrast and clear hover/focus states.
- Forms need inline error messaging and accessible labels.

## Accessibility Baseline

- Maintain WCAG AA contrast in both light and dark modes.
- Use semantic headings, landmark regions, keyboard-accessible menus, and descriptive alt text.
- Portfolio filtering, if implemented, must remain usable without pointer-only interactions.
