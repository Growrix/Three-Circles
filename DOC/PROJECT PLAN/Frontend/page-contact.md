# Contact Us Page

## Page Goal

Provide the shortest path from interest to inquiry while still collecting enough structured information to qualify the lead.

## Required Sections

1. Intro copy setting response expectations.
2. Contact form with name, email, phone, project type, service interest, timeline, budget range, message, and preferred contact method as assumed structured inputs.
3. Direct contact details block for users who prefer email or phone.
4. Optional map or service-area summary if a physical studio or coverage area is confirmed later.

## Interaction Rules

- The form must show inline validation before submission and clear server-result feedback after submission.
- Submit action must be disabled during in-flight submission.
- Success state should confirm receipt and expected follow-up window.
- Failure state should preserve entered values and provide retry guidance.

## Accessibility Notes

- Every field requires a visible label.
- Error text must be tied to inputs programmatically.
- The submission result must be announced to assistive technologies.
