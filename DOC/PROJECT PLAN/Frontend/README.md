# Frontend

## Purpose

This folder converts the shared contracts into a brand-consistent public website experience for Three Circles. It covers layout, content hierarchy, reusable UI patterns, and per-page behavior for the required public routes.

## Read Order

1. `ai-context.yaml`
2. `frontend-architecture.md`
3. `component-and-styling-standards.md`
4. `page-home.md`
5. `page-about.md`
6. `page-services.md`
7. `page-portfolio.md`
8. `page-contact.md`

## Frontend Scope

- The public experience is the current confirmed product surface.
- The website must reflect the brand voice and color guidance already captured in the source plan.
- Contact form UX may guide users through structured inputs, but server-side validation and submission states remain backend-owned.

## Key Constraints

- Do not introduce hidden routes, dashboards, or authenticated customer experiences as confirmed requirements.
- Do not invent backend permissions, payloads, or lifecycle states.
- When assumptions are necessary, keep them visibly labeled and consistent with shared contracts.
