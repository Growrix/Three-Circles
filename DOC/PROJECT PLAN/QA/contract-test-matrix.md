# Contract Test Matrix

| Contract Area | Verification Focus |
| --- | --- |
| Shared routes | Required routes match public navigation and deployment routing |
| Service offering schema | Public service data matches canonical fields and order |
| Portfolio project schema | Only published records are public and slug routing resolves correctly |
| Contact submission contract | Required and optional fields validate correctly and durable receipt is returned |
| Lead inquiry state model | Only allowed server-side transitions occur |
| Access matrix | Anonymous users cannot access internal routes or inquiry data |

## Rule

Any contract change in Shared Contracts or API and Data must update this matrix before release.
