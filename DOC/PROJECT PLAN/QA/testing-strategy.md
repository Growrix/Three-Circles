# Testing Strategy

## Test Layers

- Content rendering tests for required routes and published-state behavior.
- Component and interaction tests for navigation, portfolio cards, and contact form validation.
- API contract tests for public content endpoints and `POST /api/contact`.
- Integration tests for inquiry persistence and notification retry behavior.
- End-to-end tests for the full public journey from browsing to successful inquiry submission.

## Non-Functional Coverage

- Responsive behavior across common mobile and desktop breakpoints.
- Accessibility checks for navigation, forms, contrast, and semantic structure.
- Performance checks for homepage, services, portfolio, and contact routes.
